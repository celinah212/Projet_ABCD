A propos du projet

Objet du projet

Cette application est un systeme complet de gestion de produits qui implemente les quatre operations fondamentales (creation, lecture, modification et suppression) sur des articles references en base de donnees. Elle permet a un utilisateur de maintenir un catalogue structure par categories, de suivre l'etat du stock et la valeur globale des produits.

Architecture technique

Le backend est construit avec le framework Django en suivant son architecture MVT (Model - View - Template), une variante du modele MVC adaptee a Django. Le modele decrit les donnees, la vue traite la logique metier, et le template s'occupe du rendu HTML cote serveur.

Frontend

Le frontend est entierement realise en HTML, CSS et JavaScript standard. Aucune librairie, aucun framework et aucune precompilation ne sont utilises : pas de TypeScript, pas de TSX, pas de JSX. Le code JavaScript ajoute des interactions telles que la recherche instantanee, la bascule du theme clair/sombre et la confirmation des operations sensibles.

Donnees et stockage

La base de donnees utilisee par defaut est SQLite, qui ne necessite aucune installation supplementaire. Deux modeles sont definis : Categorie (pour structurer le catalogue) et Produit (nom, description, prix, quantite, disponibilite, dates de creation et de modification).

Fonctionnalites cles

  • Liste des produits avec recherche par nom et filtre par categorie.
  • Fiche detaillee pour chaque produit.
  • Formulaires de creation et de modification avec validation serveur.
  • Suppression d'un produit avec double confirmation.
  • Bascule entre mode clair et mode sombre, memorisee localement.
  • Option de reinitialisation complete de la base de donnees.

Perimetre

L'application n'integre aucun systeme d'authentification : elle est destinee a un usage local, mono-utilisateur, a des fins pedagogiques ou de demonstration.
